What is git
-----------
Git is a distributed version control system (DVCS) used for tracking changes in code and collaborating with multiple developers efficiently. 
It was created by Linus Torvalds in 2005 for managing the development of the Linux kernel.

Key Features of Git:
------------------
Distributed System â€“ Every developer gets a full copy of the repository, allowing offline work.
Branching & Merging â€“ Allows developers to create separate branches for features, bug fixes, or experiments and merge them when ready.
Speed & Efficiency â€“ Designed for high performance with fast commits, branching, and merging.
Security â€“ Uses cryptographic hashing (SHA-1) to ensure integrity and prevent data loss.
Collaboration â€“ Enables multiple developers to work on the same project without conflicts.
Rollback & History Tracking â€“ Keeps a history of all changes, making it easy to revert to previous versions if needed.

Difference Between git log and git reflog:
Purpose:

Scope:

Visibility:

 git reflog: Includes even deleted or lost commits (until garbage collection)

Usage:

 git reflog: Used to recover lost commits or track changes to HEAD
